# 知识组织与检索联调说明

本文档用于说明成员 B 已提供的分类、标签、知识候选召回接口，以及搜索、筛选、高亮、最近搜索等前端组件如何接入。

## 1. 当前能力边界

已实现：

- 分类 CRUD：`/api/categories`
- 标签 CRUD：`/api/tags`
- Agent 问答前候选知识召回：`POST /api/knowledge/retrieve`
- 前端可复用组件：分类管理、标签管理、搜索框、筛选栏、高亮展示、最近搜索记录

暂不提供：

- 独立的真实 LLM rerank API。当前规则精排已经内嵌在 RAG 检索链路中，由 `POST /api/knowledge/retrieve` 内部调用。
- `GET /api/knowledge/search` 的完整知识列表搜索接口。该接口依赖正式知识主表、知识列表页字段和权限规则，建议后续联调时再接入。

## 2. 通用响应格式

分类和标签接口使用统一响应结构：

```ts
type ApiSuccess<T> = {
  success: true;
  data: T;
};

type ApiFailure = {
  success: false;
  error: {
    code: string;
    message: string;
  };
};
```

常见错误：

- `VALIDATION_ERROR`：参数不合法，HTTP 400
- `CATEGORY_CONFLICT` / `TAG_CONFLICT`：名称重复，HTTP 409
- `CATEGORY_NOT_FOUND` / `TAG_NOT_FOUND`：资源不存在，HTTP 404
- `INTERNAL_ERROR`：服务端异常，HTTP 500

## 3. 分类接口

分类初版是全局共享分类，暂不绑定具体知识库或知识条目。

### 3.1 分类数据结构

```ts

  color: string | null;

  updatedAt: string;
```



## 5. 候选知识召回接口

该接口是业务侧入口，会复用已有 RAG 检索链路：

```text
query rewrite -> vector + BM25 + exact -> RRF -> rules rerank -> minScore/fallback -> MMR -> 邻近 chunk 扩展 -> context builder
```

### 5.1 请求地址

```http
POST /api/knowledge/retrieve
Content-Type: application/json
```

### 5.2 请求体

```ts
type RagRetrieveRequest = {
  query: string;
  scope: {
    knowledgeBaseIds: string[];
    knowledgeIds?: string[];
    categoryIds?: string[];
    tagIds?: string[];
    chunkTypes?: Array<"text" | "wiki" | "summary" | "qa">;
  };
  mode?: "fast" | "balanced" | "detailed";
};
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `query` | `string` | 是 | 用户问题，trim 后不能为空 |
| `scope.knowledgeBaseIds` | `string[]` | 是 | 至少一个知识库 id |
| `scope.knowledgeIds` | `string[]` | 否 | 限定知识条目范围 |
| `scope.categoryIds` | `string[]` | 否 | 限定分类范围 |
| `scope.tagIds` | `string[]` | 否 | 限定标签范围 |
| `scope.chunkTypes` | `string[]` | 否 | 限定 chunk 类型 |
| `mode` | `fast \| balanced \| detailed` | 否 | 默认 `balanced` |

请求示例：

```json
{
  "query": "普通成员可以删除知识库吗？",
  "scope": {
    "knowledgeBaseIds": ["kb_001"],
    "categoryIds": ["cat_permission"],
    "tagIds": ["tag_member", "tag_permission"],
    "chunkTypes": ["text", "wiki", "qa"]
  },
  "mode": "balanced"
}
```

### 5.3 成功响应

`POST /api/knowledge/retrieve` 成功时直接返回 RAG 响应体，不再包一层 `success`。

```ts
type RagRetrieveResponse = {
  query: string;
  contexts: RagContext[];
  llmContext: string;
  references: RagReference[];
  metrics?: {
    fallback: boolean;
    fallbackReason?: "below_min_score_but_keep_top1" | "no_relevant_context";
    minScore: number;
    fallbackTop1Score: number;
    beforeMinScoreCount: number;
    afterMinScoreCount: number;
    topScore?: number;
  };
};

type RagContext = {
  id: string;
  knowledgeBaseId: string;
  knowledgeId: string;
  chunkId: string;
  title: string;
  content: string;
  chunkType: "text" | "wiki" | "summary" | "qa";
  score: number;
  categoryId?: string;
  tagIds?: string[];
  metadata?: Record<string, unknown>;
};

type RagReference = {
  refId: string;
  knowledgeBaseId: string;
  knowledgeId: string;
  chunkId: string;
  title: string;
  chunkType: "text" | "wiki" | "summary" | "qa";
};
```

### 5.5 筛选值转换为 RAG scope

知识筛选栏中的值可以这样转换给召回接口：

```ts
import type { KnowledgeFilterValue } from "@/features/knowledge";

function toRetrieveScope(
  knowledgeBaseIds: string[],
  filters: KnowledgeFilterValue
) {
  return {
    knowledgeBaseIds,
    categoryIds: filters.categoryId ? [filters.categoryId] : undefined,
    tagIds: filters.tagIds.length > 0 ? filters.tagIds : undefined,
  };
}
```

调用示例：

```ts
const response = await fetch("/api/knowledge/retrieve", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    query: keyword,
    scope: toRetrieveScope(["kb_001"], filters),
    mode: "balanced",
  }),
});

const result = await response.json();
```

## 8. 搜索框组件

`KnowledgeSearchBox` 是受控组件，只负责输入、清空、提交和可选防抖通知，不直接请求接口。

```tsx
"use client";

import { useState } from "react";
import { KnowledgeSearchBox } from "@/features/knowledge";

export function KnowledgeListSearch() {
  const [keyword, setKeyword] = useState("");
  const [loading, setLoading] = useState(false);

  async function fetchKnowledgeList(nextKeyword: string) {
    setLoading(true);
    try {
      // 后续接入正式 GET /api/knowledge/search 或页面自己的列表查询方法
      console.log("search keyword:", nextKeyword);
    } finally {
      setLoading(false);
    }
  }

  return (
    <KnowledgeSearchBox
      value={keyword}
      loading={loading}
      placeholder="搜索知识标题、摘要或正文关键词"
      debounceMs={300}
      onChange={setKeyword}
      onSearch={(value) => void fetchKnowledgeList(value)}
      onClear={() => void fetchKnowledgeList("")}
      onDebouncedChange={(value) => {
        console.log("debounced keyword:", value);
      }}
    />
  );
}
```

组件参数：

```ts
export type KnowledgeSearchBoxProps = {
  value: string;
  placeholder?: string;
  disabled?: boolean;
  loading?: boolean;
  autoFocus?: boolean;
  maxLength?: number;
  debounceMs?: number;
  onChange: (value: string) => void;
  onSearch?: (keyword: string) => void;
  onClear?: () => void;
  onDebouncedChange?: (keyword: string) => void;
};
```

注意：

- 点击搜索按钮或按 Enter 会触发 `onSearch(trimmedKeyword)`。
- 点击清空会触发 `onChange("")` 和 `onClear()`。
- `debounceMs` 只负责通知 `onDebouncedChange`，组件不会自动请求接口。

## 9. 筛选栏组件

`KnowledgeFilterBar` 是受控组件，分类和标签数据由调用方传入。

```tsx
"use client";

import { useEffect, useState } from "react";
import {
  KnowledgeFilterBar,
  type KnowledgeCategoryDto,
  type KnowledgeFilterValue,
  type KnowledgeTagDto,
} from "@/features/knowledge";
import { fetchCategories } from "@/features/knowledge/api/categories";
import { fetchTags } from "@/features/knowledge/api/tags";

export function KnowledgeListFilters() {
  const [categories, setCategories] = useState<KnowledgeCategoryDto[]>([]);
  const [tags, setTags] = useState<KnowledgeTagDto[]>([]);
  const [filters, setFilters] = useState<KnowledgeFilterValue>({
    tagIds: [],
  });

  useEffect(() => {
    void Promise.all([fetchCategories(), fetchTags()]).then(
      ([nextCategories, nextTags]) => {
        setCategories(nextCategories);
        setTags(nextTags);
      }
    );
  }, []);

  return (
    <KnowledgeFilterBar
      value={filters}
      categories={categories}
      tags={tags}
      onChange={(nextFilters) => {
        setFilters(nextFilters);
        console.log("filters changed:", nextFilters);
      }}
      onReset={() => {
        console.log("filters reset");
      }}
    />
  );
}
```

筛选值类型：

```ts
export type KnowledgeStatusFilter = "available" | "pending" | "disabled";

export type KnowledgeFilterValue = {
  categoryId?: string;
  tagIds: string[];
  status?: KnowledgeStatusFilter;
};
```

状态含义：

| 值 | 展示文案 | 说明 |
| --- | --- | --- |
| `available` | 可用 | 可被管理端展示和 Agent 检索 |
| `pending` | 待确认 | 待审核或待确认 |
| `disabled` | 不可用 | 不应返回给 Agent |
| `undefined` | 全部状态 | 不按状态过滤 |

### 9.1 转换为知识列表查询参数

后续正式知识列表接口可以参考以下 query 参数格式：

```ts
function buildKnowledgeSearchParams(
  keyword: string,
  filters: KnowledgeFilterValue
) {
  const params = new URLSearchParams();
  const trimmedKeyword = keyword.trim();

  if (trimmedKeyword) params.set("keyword", trimmedKeyword);
  if (filters.categoryId) params.set("categoryId", filters.categoryId);
  if (filters.tagIds.length > 0) params.set("tagIds", filters.tagIds.join(","));
  if (filters.status) params.set("status", filters.status);

  return params;
}
```

## 12. 最近搜索记录

最近搜索记录只保存在浏览器 `localStorage`，不写数据库。

### 12.1 Hook

```tsx
import { useRecentKnowledgeSearches } from "@/features/knowledge";

const recent = useRecentKnowledgeSearches({
  storageKey: "rag-creater:knowledge:recent-searches",
  maxItems: 8,
});

recent.addSearch("权限管理");
recent.removeSearch("权限管理");
recent.clearSearches();
```

Hook 返回值：

```ts
{
  searches: Array<{
    keyword: string;
    searchedAt: string;
  }>;
  addSearch: (keyword: string) => void;
  removeSearch: (keyword: string) => void;
  clearSearches: () => void;
}
```

规则：

- `addSearch` 会 trim，空字符串不保存。
- 重复关键词会去重，并移动到第一位。
- 默认最多保留 8 条。
- localStorage 不可用时静默降级为空列表。

### 12.2 组件

```tsx
import { RecentSearches } from "@/features/knowledge";

<RecentSearches
  searches={recent.searches}
  onSelect={(value) => {
    setKeyword(value);
    void fetchKnowledgeList(value, filters);
  }}
  onRemove={recent.removeSearch}
  onClear={recent.clearSearches}
/>;
```

组件参数：

```ts
export type RecentSearchesProps = {
  searches: RecentKnowledgeSearch[];
  disabled?: boolean;
  title?: string;
  emptyText?: string;
  onSelect: (keyword: string) => void;
  onRemove?: (keyword: string) => void;
  onClear?: () => void;
};
```

```tsx
  categoryId?: string | null;
  tagIds?: string[];
  status: "available" | "pending" | "disabled";
};

export function KnowledgeListPanel() {
  const [keyword, setKeyword] = useState("");
  const [filters, setFilters] = useState<KnowledgeFilterValue>({ tagIds: [] });
  const [categories, setCategories] = useState<KnowledgeCategoryDto[]>([]);
  const [tags, setTags] = useState<KnowledgeTagDto[]>([]);
  const [items, setItems] = useState<KnowledgeListItem[]>([]);
  const [loading, setLoading] = useState(false);
  const recent = useRecentKnowledgeSearches();

  useEffect(() => {
    void Promise.all([fetchCategories(), fetchTags()]).then(
      ([nextCategories, nextTags]) => {
        setCategories(nextCategories);
        setTags(nextTags);
      }
    );
  }, []);

  async function fetchKnowledgeList(
    nextKeyword: string,
    nextFilters: KnowledgeFilterValue
  ) {
    setLoading(true);
    try {
      const params = buildKnowledgeSearchParams(nextKeyword, nextFilters);
      // 等正式知识搜索接口落地后替换这里
      const response = await fetch(`/api/knowledge/search?${params}`);
      const json = await response.json();
      setItems(json.data ?? []);
    } finally {
      setLoading(false);
    }
  }

  return (
    <div className="space-y-4">
      <KnowledgeSearchBox
        value={keyword}
        loading={loading}
        onChange={setKeyword}
        onSearch={(value) => {
          recent.addSearch(value);
          void fetchKnowledgeList(value, filters);
        }}
        onClear={() => void fetchKnowledgeList("", filters)}
      />

      <RecentSearches
        searches={recent.searches}
        onSelect={(value) => {
          setKeyword(value);
          void fetchKnowledgeList(value, filters);
        }}
        onRemove={recent.removeSearch}
        onClear={recent.clearSearches}
      />

      <KnowledgeFilterBar
        value={filters}
        categories={categories}
        tags={tags}
        onChange={(nextFilters) => {
          setFilters(nextFilters);
          void fetchKnowledgeList(keyword, nextFilters);
        }}
      />

      <div className="space-y-3">
        {items.map((item) => (
          <article key={item.id} className="rounded-lg border border-zinc-200 p-4">
            <h3 className="font-medium text-zinc-900">
              <SearchHighlight text={item.title} keyword={keyword} />
            </h3>
            <p className="mt-2 text-sm text-zinc-500">
              <SearchHighlight
                text={item.summary ?? ""}
                keyword={keyword}
                maxLength={120}
                emptyText="暂无摘要"
              />
            </p>
          </article>
        ))}
      </div>
    </div>
  );
}

function buildKnowledgeSearchParams(
  keyword: string,
  filters: KnowledgeFilterValue
) {
  const params = new URLSearchParams();
  const trimmedKeyword = keyword.trim();

  if (trimmedKeyword) params.set("keyword", trimmedKeyword);
  if (filters.categoryId) params.set("categoryId", filters.categoryId);
  if (filters.tagIds.length > 0) params.set("tagIds", filters.tagIds.join(","));
  if (filters.status) params.set("status", filters.status);

  return params;
}
```

## 14. 与知识入库/增删改的联调点

RAG 检索模块已经有本地向量索引持久化能力。后续知识管理或入库模块在知识 chunk 变化时，建议调用以下方法维护索引：

```ts
import {
  deleteChunkEmbedding,
  indexChunk,
  indexChunks,
  reindexOutdatedChunks,
} from "@/server/services/rag/vector-index-repository";

// 新增或更新单个 chunk 后
await indexChunk(chunk);

// 批量导入 chunk 后
await indexChunks(chunks);

// 删除 chunk 后
await deleteChunkEmbedding(chunkId);

// 批量检查内容 hash，过期才重建 embedding
await reindexOutdatedChunks(chunks);
```

注意：

- 当前 embedding 仍是 mock/hash embedding，不接真实 embedding API。
- 如果调用方不主动建索引，vector search 发现缺失或过期时也会兜底生成并写入索引。
- 删除 chunk 时需要由知识管理模块主动调用 `deleteChunkEmbedding(chunkId)`，否则本地向量索引表中可能保留无效记录。

## 15. 联调注意事项

- 分类和标签是全局共享数据，当前没有知识库隔离。
- 分类和标签暂未绑定知识条目，正式知识主表稳定后再建立绑定关系或由知识条目字段引用 `categoryId/tagIds`。
- `KnowledgeFilterBar` 不会请求分类/标签数据，页面需要先调用 `fetchCategories/fetchTags` 后传入。
- `KnowledgeSearchBox` 不会自动请求接口，页面需要在 `onSearch` 或 `onDebouncedChange` 中触发查询。
- `SearchHighlight` 只做展示高亮，不负责筛选数据。
- `RecentSearches` 只读写浏览器 localStorage，不会同步到服务端。
- `POST /api/knowledge/retrieve` 只返回候选上下文和引用，不负责生成最终回答。
- Agent 侧只应使用 `available` 状态的知识，后端 RAG 检索链路也会过滤不可用 chunk。
