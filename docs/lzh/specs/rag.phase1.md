# RAG 知识库管理模块 Phase 1 Spec

## 1. 阶段目标

Phase 1 用于实现 RAG 知识库管理页面的基础管理能力，范围限定为：

- 知识库列表展示
- 搜索、排序、排序倒置
- 统计卡片与状态快捷筛选
- 新建知识库
- 编辑知识库
- 删除知识库
- 前端 mock 数据
- Zustand 全局状态共享
- 前端数据兜底

文档管理、文件上传、知识分片查看等能力不在 Phase 1 实现，拆分到 `docs/lzh/specs/rag.pase2.md`。

当前阶段以前端页面和交互闭环为主。页面初始化需要模拟“向后端请求知识库列表”的流程；新建、编辑、删除暂不落库，只更新前端状态。后续接入真实接口时，前端仍需要保留数据兜底逻辑，避免接口字段缺失或类型异常导致页面崩溃。

---

## 2. 布局与路由依赖

本模块在管理后台整体布局内渲染。

通用布局、Header、Sidebar、Main Content、侧边栏导航、预留页面和 `/` 到 `/knowledge-bases` 的默认重定向要求，统一参考：

- `docs/lzh/specs/layoutspec.md`
- `docs/lzh/tecdoc/layout.md`

本 spec 只描述 `/knowledge-bases` 知识库管理页面在 Phase 1 的业务内容和交互，不重复定义通用布局。

## 3. 数据与状态要求

### 3.1 mock 数据位置

Phase 1 的 mock 数据应放在：

```text
src/features/knowledge-bases/mock-data.ts
```

mock 数据用于初始化知识库列表，并为后续页面共享提供基础数据。

mock 初始数据要求：

- 初始化 3-5 条知识库。
- 同时覆盖 `active` 和 `disabled` 两种状态。
- 覆盖不同文档数量和 Chunks 数量，用于展示统计与排序效果。
- mock 列表数据只需要包含知识库卡片展示所需信息和统计字段，不需要包含 documents 和 chunks 明细。
- mock 数据不需要模拟真实接口延迟。

### 3.2 Zustand 全局状态

知识库数据需要接入 Zustand 全局状态，状态定义和操作方法放在：

```text
src/store/slices/knowledge-base-slice.ts
```

该文件当前为模板代码，Phase 1 实现时可以按本 spec 调整。

状态要求：

- 存放知识库列表。
- 支持新增知识库。
- 支持编辑知识库。
- 支持删除知识库。
- 支持设置当前选中的知识库 id。
- 可预留 `selected`、`selectedDocs`、`selectedChunks` 等详情状态，但 Phase 1 页面暂不消费文档和分片详情。
- 支持保存 loading 和 error 状态。
- 状态字段命名保持简洁，并需要有简洁注释说明用途。

Phase 1 只有初始化列表请求模拟后端流程；新建、编辑、删除暂不落库，只更新前端 Zustand。后续接入真实 API 时再替换为 POST / PATCH / DELETE。

### 3.3 共享数据原则

知识库列表后续会被多个页面或弹窗共享，因此知识库列表数据不应只放在页面组件本地 state。

Phase 1 主列表只维护轻量知识库数据。文档列表和分片明细属于 Phase 2 范围，后续只在用户选中并查看某个知识库时按需加载。

页面内部仍可使用本地 state 管理纯 UI 状态，例如：

- 搜索输入框当前值
- 已提交的搜索关键词
- 当前排序字段
- 当前排序方向
- 当前统计卡片筛选状态
- 当前打开的弹窗类型
- 当前编辑中的表单草稿

## 4. 前端展示模型

数据库结构参考：

- `docs/DataType.md`
- `prisma/schema.prisma`

知识库状态统一使用数据库字段值：

| 字段值 | 页面展示 |
| --- | --- |
| `active` | 启用 |
| `disabled` | 禁用 |

Phase 1 前端展示字段建议如下：

```ts
type KnowledgeBaseStatus = "active" | "disabled";

type KnowledgeBaseViewModel = {
  id: string;
  name: string;
  description: string;
  icon?: string;
  documentCount: number;
  chunkCount: number;
  topK: number;
  chunkSize: number;
  similarityThreshold: number;
  status: KnowledgeBaseStatus;
  updatedAt: string;
};
```

说明：

- `documentCount` 和 `chunkCount` 是列表展示所需统计字段。
- Phase 1 列表数据不包含 `documents` 和 `chunks` 明细。
- 文档和分片详情放到 Phase 2，在用户查看某个知识库时按需加载。

## 5. 页面结构

知识库管理页面从上到下分为三部分：

1. 筛选操作栏
2. 数据统计卡片区
3. 知识库卡片展示区

---

## 6. 筛选操作栏

筛选操作栏位于页面顶部，包含四个控件：

1. 搜索框
2. 排序下拉框
3. 排序倒置按钮
4. 新建知识库按钮

UI 风格要求：

- 表单、弹窗、选择器、开关等交互组件优先使用 shadcn/ui 风格组件。
- 可以根据实现需要自由新增 shadcn/ui 组件，保持后台页面风格一致。

### 6.1 搜索框

用户可以通过输入关键词搜索知识库。

交互要求：

- 输入过程中不实时过滤。
- 用户输入关键词后，按 `Enter` 键提交搜索关键词。
- 搜索结果只展示符合搜索条件的知识库。
- 搜索范围至少包括知识库名称和知识库描述。
- 当提交的搜索关键词为空时，展示全部知识库。
- 清空搜索输入后，立即清空已提交搜索关键词，并展示全部知识库。

### 6.2 排序下拉框

下拉框仅用于切换知识库排序方式，不负责状态筛选。

选项包括：

| 选项 | 含义 |
| --- | --- |
| 更新时间 | 按最后更新时间排序 |
| 包含文档数量 | 按文档数量排序 |

默认排序规则：

- 默认排序字段为 `更新时间`。
- 默认排序方向为倒序，即最新的知识库排在前面。

状态筛选由统计卡片负责，下拉框与统计卡片不联动。

### 6.3 排序倒置按钮

点击排序倒置按钮后，当前知识库卡片展示顺序反转。

要求：

- 每次点击切换一次排序方向。
- 可用于反转当前搜索、排序和统计卡片筛选后的展示结果。
- 推荐使用图标按钮表示，例如上下箭头、排序图标等。

### 6.4 新建知识库按钮

新建按钮用于打开新建知识库弹窗。

交互要求：

- 点击后打开 `新建知识库弹窗`。
- 弹窗用于创建新的知识库。
- 当前阶段使用 Zustand 状态模拟创建结果。

---

## 7. 数据统计卡片区

筛选操作栏下方展示三个数据统计卡片：

| 卡片 | 展示内容 | Icon 颜色 |
| --- | --- | --- |
| 知识库总量 | 当前知识库总数量 | 蓝色 |
| 启用知识库数量 | status 为 active 的知识库数量 | 绿色 |
| 禁用知识库数量 | status 为 disabled 的知识库数量 | 红色 |

统计卡片数量始终按全部知识库计算，与搜索关键词、排序方式和当前卡片筛选状态无关。

### 7.1 统计卡片点击筛选

三个统计卡片同时具备快捷筛选能力。

点击 `知识库总量`：

- 高亮总量卡片。
- 展示全部知识库。
- 不改变排序下拉框当前选项。

点击 `启用知识库数量`：

- 高亮启用卡片。
- 只展示启用状态知识库。
- 不改变排序下拉框当前选项。

点击 `禁用知识库数量`：

- 高亮禁用卡片。
- 只展示禁用状态知识库。
- 不改变排序下拉框当前选项。

再次点击已高亮卡片：

- 取消当前卡片高亮。
- 展示全部知识库。
- 不改变排序下拉框当前选项。

---

## 8. 知识库展示卡片

知识库默认以卡片形式排列展示。

每个知识库卡片展示以下信息：

| 区域 | 内容 |
| --- | --- |
| 左上角 | 知识库 Logo / Icon |
| 标题区域 | 知识库名称 |
| 描述区域 | 知识库描述，使用较小字号 |
| 指标区域 | 文档数量、Chunks 数量、TopK、相似度阈值 |
| 时间区域 | 最后更新时间 |
| 右上角 | 知识库状态 |
| 底部操作区 | 编辑、删除按钮 |

Phase 1 不实现 `查看知识` 操作按钮。该能力属于 Phase 2。

### 8.1 状态展示

| 状态 | 展示文字 | 样式 |
| --- | --- | --- |
| active | 启用 | 绿色、加粗 |
| disabled | 禁用 | 红色、加粗 |

### 8.2 指标展示

知识库卡片需要展示：

- 文档数量，使用列表数据中的 `documentCount`。
- Chunks 数量，使用列表数据中的 `chunkCount`。
- TopK 值。
- 相似度得分阈值。
- 最后更新时间。

字段兜底：

| 字段 | 兜底值 |
| --- | --- |
| 知识库名称 | `未命名知识库` |
| 描述 | `暂无描述` |
| 文档数量 | `0` |
| Chunks 数量 | `0` |
| TopK | `0` |
| 相似度得分阈值 | `0` |
| 最后更新时间 | `--` |
| status | `disabled` |

## 9. 新建和编辑知识库弹窗

点击 `新建知识库` 按钮后，打开新建知识库弹窗。

点击知识库卡片中的 `编辑` 按钮后，打开编辑知识库弹窗。

新建弹窗和编辑弹窗 UI 与字段保持一致。

### 9.1 可编辑字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| name | string | 知识库名称 |
| description | string | 知识库描述 |
| topK | number | 检索时返回的 TopK 数量 |
| similarityThreshold | number | 相似度得分阈值 |
| chunkSize | number | 分片大小 |
| status | active / disabled | 是否启用 |

### 9.2 新建默认值

新建弹窗初始字段：

- `name` 为空。
- `description` 为空。
- `topK` 默认值为 `5`。
- `chunkSize` 默认值为 `500`。
- `similarityThreshold` 默认值为 `0.7`。
- `status` 默认值为 `active`。

### 9.3 表单校验

保存前必须校验：

- `name` 必填。
- `topK` 必须为正整数。
- `chunkSize` 必须为正整数。
- `similarityThreshold` 范围为 `0-1`。
- 新建时，名称不能与任意已有知识库重名。
- 编辑时，名称重复校验需要排除当前编辑知识库 id。

校验失败时：

- 不能保存。
- 在保存按钮上方展示一行红色错误提示，说明失败原因。

### 9.4 状态切换

状态字段使用滑动开关实现。

- 开启时表示 `启用`。
- 关闭时表示 `禁用`。

### 9.5 弹窗操作

弹窗底部包含：

- 取消按钮
- 确认按钮

点击取消：

- 关闭弹窗。
- 不保存修改。

点击确认：

- 校验表单。
- 校验通过后写入 Zustand 知识库状态。
- 新建成功后，将 `updatedAt` 设置为当前时间。
- 编辑成功后，将 `updatedAt` 更新为当前时间。
- 关闭弹窗。
- 更新对应知识库卡片展示。

---

## 10. 删除知识库确认框

点击知识库卡片中的 `删除` 按钮后，弹出删除确认框。

删除确认框文案要求：

- 标题为 `确认删除知识库`。
- 正文需要包含将被删除的知识库名称。
- 确认按钮文案为 `确认删除`。

### 10.1 关闭方式

以下操作均视为取消删除：

- 点击右上角关闭图标。
- 点击取消按钮。
- 点击弹窗外部遮罩区域。

### 10.2 确认删除

只有点击确认按钮时，才真正删除该知识库。

删除后：

- 当前知识库卡片从页面中移除。
- 统计卡片数量同步更新。
- Zustand 知识库状态同步更新。

## 11. 前端数据兜底要求

### 11.1 列表数据兜底

如果接口返回或初始化数据不是数组，则使用空数组：

```ts
const safeKnowledgeBases = Array.isArray(data) ? data : [];
```

### 11.2 字段兜底

渲染知识库卡片时，应对字段进行兜底：

```ts
const name = typeof item.name === "string" ? item.name : "未命名知识库";
const description =
  typeof item.description === "string" ? item.description : "暂无描述";
const documentCount =
  typeof item.documentCount === "number" ? item.documentCount : 0;
const chunkCount = typeof item.chunkCount === "number" ? item.chunkCount : 0;
const topK = typeof item.topK === "number" ? item.topK : 0;
const similarityThreshold =
  typeof item.similarityThreshold === "number" ? item.similarityThreshold : 0;
const status =
  item.status === "active" || item.status === "disabled"
    ? item.status
    : "disabled";
```

## 12. 空状态与异常状态

### 12.1 知识库为空

当知识库列表为空时，展示空状态。

空状态内容建议：

- 文案：`暂无知识库`
- 辅助文案：`点击新建按钮创建你的第一个知识库`
- 操作按钮：`新建知识库`

### 12.2 搜索无结果

当搜索或筛选后无结果时，展示搜索空状态。

建议文案：

- `没有找到符合条件的知识库`

### 12.3 数据异常

当数据结构异常时：

- 页面不崩溃。
- 使用空数组或字段默认值兜底。
- 可以在控制台输出 warning，方便开发调试。

## 14. Phase 1 验收标准

1. 知识库管理页面能够在管理后台 Main Content 中渲染。
2. 知识库页面顶部包含搜索框、排序下拉框、排序倒置按钮和新建按钮。
3. 搜索框支持输入关键词后按 Enter 搜索。
4. 输入过程中不实时过滤。
5. 清空搜索输入后立即展示全部知识库。
6. 排序下拉框支持 `更新时间` 和 `包含文档数量` 两个选项。
7. 默认按更新时间倒序排序，最新知识库在前。
8. 排序倒置按钮可以反转当前卡片展示顺序。
9. 页面展示知识库总量、启用数量、禁用数量三个统计卡片。
10. 统计卡片数量始终按全部知识库计算，不受搜索和排序影响。
11. 点击统计卡片可以快捷筛选全部、启用或禁用知识库。
12. 再次点击已高亮统计卡片可以取消筛选。
13. 知识库以卡片形式展示。
14. 每个知识库卡片展示名称、描述、图标、文档数量、Chunks 数量、TopK、相似度阈值、更新时间和状态。
15. 文档数量使用列表数据中的 `documentCount`。
16. Chunks 数量使用列表数据中的 `chunkCount`。
17. 知识库状态以 `active` 绿色启用、`disabled` 红色禁用展示。
18. 点击新建按钮可以打开新建知识库弹窗。

22. 新建知识库默认 `topK` 为 `5`。
23. 新建知识库默认 `chunkSize` 为 `500`。

25. 新建时不能与任意知识库重名。
26. 编辑时名称重复校验排除当前知识库 id。
27. 新建成功后 `updatedAt` 设置为当前时间。
28. 编辑成功后 `updatedAt` 更新为当前时间。
29. 表单校验失败时不能保存，并在保存按钮上方展示红色错误提示。
30. 点击知识库删除按钮会弹出确认框。
31. 删除确认框标题为 `确认删除知识库`。
32. 删除确认框正文包含将被删除的知识库名称。
33. 删除确认框确认按钮文案为 `确认删除`。
34. 只有点击确认按钮才会真正删除知识库。
35. 关闭图标、取消按钮、点击弹窗外部均不会删除知识库。
36. 新建、编辑、删除操作会同步更新 Zustand 知识库状态。

38. mock 初始数据包含 3-5 条知识库。
