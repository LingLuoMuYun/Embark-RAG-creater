# RAG 知识库管理模块 Phase 1 Spec

## 1. 阶段目标

Phase 1 用于实现 RAG 知识库管理页面的基础管理能力，范围限定为：

- 知识库列表展示
- 搜索、排序、排序倒置
- 统计卡片与状态快捷筛选
- 新建知识库
- 编辑知识库
- 删除知识库
- 前端 mock 数据
- Zustand 全局状态共享
- 前端数据兜底

文档管理、文件上传、知识分片查看等能力不在 Phase 1 实现，拆分到 `docs/lzh/specs/rag.phase2.md`。

当前阶段以前端页面和交互闭环为主，不接入真实后端接口。后端接口后续统一接入时，前端仍需要保留数据兜底逻辑，避免接口字段缺失或类型异常导致页面崩溃。

---

## 2. 布局与路由依赖

本模块在管理后台整体布局内渲染。

通用布局、Header、Sidebar、Main Content、侧边栏导航、预留页面和 `/` 到 `/knowledge-bases` 的默认重定向要求，统一参考：

- `docs/lzh/specs/layoutspec.md`
- `docs/lzh/tecdoc/layout.md`

本 spec 只描述 `/knowledge-bases` 知识库管理页面在 Phase 1 的业务内容和交互，不重复定义通用布局。

## 3. 数据与状态要求

### 3.1 mock 数据位置

Phase 1 的 mock 数据应放在：

```text
src/features/knowledge-bases/mock-data.ts
```

mock 数据用于初始化知识库列表，并为后续页面共享提供基础数据。

### 3.2 Zustand 全局状态

知识库数据需要接入 Zustand 全局状态，状态定义和操作方法放在：

```text
src/store/slices/knowledge-base-slice.ts
```

状态要求：

- 存放知识库列表。
- 支持新增知识库。
- 支持编辑知识库。
- 支持删除知识库。
- 支持设置当前选中的知识库 id。
- 支持保存 loading 和 error 状态。
- 状态字段需要有简洁注释，说明用途。

不要求在 Phase 1 接入真实接口，也不要求持久化到本地存储。

### 3.3 共享数据原则

知识库列表、文档列表和分片列表后续会被多个页面或弹窗共享，因此知识库数据不应只放在页面组件本地 state。

页面内部仍可使用本地 state 管理纯 UI 状态，例如：

- 搜索输入框当前值
- 已提交的搜索关键词
- 当前排序字段
- 当前排序方向
- 当前统计卡片筛选状态
- 当前打开的弹窗类型
- 当前编辑中的表单草稿

## 4. 前端展示模型

数据库结构参考：

- `docs/DataType.md`
- `prisma/schema.prisma`

知识库状态统一使用数据库字段值：

| 字段值     | 页面展示 |
| ---------- | -------- |
| `active`   | 启用     |
| `disabled` | 禁用     |

Phase 1 前端展示模型建议如下：

```ts
type KnowledgeBaseStatus = "active" | "disabled";
type KnowledgeBaseIcon =
  | "Database"
  | "BookOpen"
  | "FileText"
  | "Folder"
  | "Archive"
  | "Brain"
  | "Bot"
  | "GraduationCap"
  | "BriefcaseBusiness"
  | "Lightbulb";

type KnowledgeChunkViewModel = {
  id: string;
  content: string;
  tokenCount?: number;
  charCount?: number;
  createdAt?: string;
};

type KnowledgeDocumentViewModel = {
  id: string;
  name: string;
  size: number;
  uploadedAt: string;
  chunks: KnowledgeChunkViewModel[];
};

type KnowledgeBaseViewModel = {
  id: string;
  name: string;
  description: string;
  icon: KnowledgeBaseIcon;
  topK: number;
  chunkSize: number;
  similarityThreshold: number;
  status: KnowledgeBaseStatus;
  updatedAt: string;
  documents: KnowledgeDocumentViewModel[];
};
```

说明：

- `documentCount` 不作为独立持久字段，展示时从 `documents.length` 派生。
- `chunkCount` 不作为独立持久字段，展示时从 `documents[].chunks.length` 汇总派生。
- `icon` 用于控制知识库卡片左上角展示图标，值必须来自预设的十个可选图标；图标颜色由该字段同步派生，不单独暴露颜色编辑项。
- Phase 1 不展示文档管理弹窗，但数据结构保留 `documents`，用于计算文档数量和 Chunks 数量，也为 Phase 2 复用。

### 4.1 可选图标与颜色

新建和编辑知识库时，用户可以通过下拉框选择知识库图标。可选图标固定为以下十种，图标库统一使用 lucide：

| icon 字段值 | 图标含义 | 推荐颜色 |
| --- | --- | --- |
| `Database` | 数据库 | 淡蓝色 |
| `BookOpen` | 书籍/知识 | 淡绿色 |
| `FileText` | 文档 | 淡青色 |
| `Folder` | 文件夹 | 淡黄色 |
| `Archive` | 归档 | 淡橙色 |
| `Brain` | 智能/认知 | 淡紫色 |
| `Bot` | 机器人/Agent | 淡靛色 |
| `GraduationCap` | 培训/学习 | 淡粉色 |
| `BriefcaseBusiness` | 业务/企业 | 淡灰蓝色 |
| `Lightbulb` | 创意/经验 | 淡琥珀色 |

颜色要求：

- 每个可选图标需要有稳定的浅色图标色和浅色背景色。
- 知识库卡片展示时，图标颜色与 `icon` 字段同步，不允许页面随机生成颜色。
- mock 数据、新建默认值和异常兜底都应使用 `Database`。

## 5. 页面结构

知识库管理页面从上到下分为三部分：

1. 筛选操作栏
2. 数据统计卡片区
3. 知识库卡片展示区

---

## 6. 筛选操作栏

筛选操作栏位于页面顶部，包含四个控件：

1. 搜索框
2. 排序下拉框
3. 排序倒置按钮
4. 新建知识库按钮

UI 风格要求：

- 表单、弹窗、选择器、开关等交互组件优先使用 shadcn/ui 风格组件。
- 可以根据实现需要自由新增 shadcn/ui 组件，保持后台页面风格一致。

### 6.1 搜索框

用户可以通过输入关键词搜索知识库。

交互要求：

- 输入过程中不实时过滤。
- 用户输入关键词后，按 `Enter` 键提交搜索关键词。
- 搜索结果只展示符合搜索条件的知识库。
- 搜索范围至少包括知识库名称和知识库描述。
- 当提交的搜索关键词为空时，展示全部知识库。
- 清空搜索输入后，立即清空已提交搜索关键词，并展示全部知识库。

### 6.2 排序下拉框

下拉框仅用于切换知识库排序方式，不负责状态筛选。

选项包括：

| 选项         | 含义               |
| ------------ | ------------------ |
| 更新时间     | 按最后更新时间排序 |
| 包含文档数量 | 按文档数量排序     |

默认排序规则：

- 默认排序字段为 `更新时间`。
- 默认排序方向为倒序，即最新的知识库排在前面。

状态筛选由统计卡片负责，下拉框与统计卡片不联动。

### 6.3 排序倒置按钮

点击排序倒置按钮后，当前知识库卡片展示顺序反转。

要求：

- 每次点击切换一次排序方向。
- 可用于反转当前搜索、排序和统计卡片筛选后的展示结果。
- 推荐使用图标按钮表示，例如上下箭头、排序图标等。

### 6.4 新建知识库按钮

新建按钮用于打开新建知识库弹窗。

交互要求：

- 点击后打开 `新建知识库弹窗`。
- 弹窗用于创建新的知识库。
- 当前阶段使用 Zustand 状态模拟创建结果。

---

## 7. 数据统计卡片区

筛选操作栏下方展示三个数据统计卡片：

| 卡片           | 展示内容                        | Icon 颜色 |
| -------------- | ------------------------------- | --------- |
| 知识库总量     | 当前知识库总数量                | 蓝色      |
| 启用知识库数量 | status 为 active 的知识库数量   | 绿色      |
| 禁用知识库数量 | status 为 disabled 的知识库数量 | 红色      |

统计卡片数量始终按全部知识库计算，与搜索关键词、排序方式和当前卡片筛选状态无关。

统计卡片视觉要求：

- `知识库总量` 的 icon 使用蓝色系浅色样式。
- `启用知识库数量` 的 icon 使用绿色系浅色样式。
- `禁用知识库数量` 的 icon 使用红色系浅色样式。
- 三张统计卡片可以使用浅色背景或浅色 icon 容器增强层次，但整体颜色过渡应克制，避免高饱和大面积色块。

### 7.1 统计卡片点击筛选

三个统计卡片同时具备快捷筛选能力。

点击 `知识库总量`：

- 高亮总量卡片。
- 展示全部知识库。
- 不改变排序下拉框当前选项。

点击 `启用知识库数量`：

- 高亮启用卡片。
- 只展示启用状态知识库。
- 不改变排序下拉框当前选项。

点击 `禁用知识库数量`：

- 高亮禁用卡片。
- 只展示禁用状态知识库。
- 不改变排序下拉框当前选项。

再次点击已高亮卡片：

- 取消当前卡片高亮。
- 展示全部知识库。
- 不改变排序下拉框当前选项。

## 8. 知识库展示卡片

知识库默认以卡片形式排列展示。

每个知识库卡片展示以下信息：

| 区域       | 内容                                    |
| ---------- | --------------------------------------- |
| 左上角     | 知识库 Logo / Icon                      |
| 标题区域   | 知识库名称                              |
| 描述区域   | 知识库描述，使用较小字号                |
| 指标区域   | 文档数量、Chunks 数量、TopK、相似度阈值 |
| 时间区域   | 最后更新时间                            |
| 右上角     | 知识库状态                              |
| 底部操作区 | 编辑、删除按钮                          |

Phase 1 不实现 `查看知识` 操作按钮。该能力属于 Phase 2。

卡片布局与视觉要求：

- 知识库卡片列表在桌面端每行展示 3 张卡片。
- 卡片内容需要略微紧凑，避免一行三列时出现大面积空白。
- 卡片与页面底色之间需要有轻微层次差异，可通过浅色背景、边框和弱阴影实现。
- 鼠标悬停在卡片上时，卡片需要有明显但克制的状态变化，例如整体轻微放大、阴影增强或边框颜色变化。
- `编辑` 和 `删除` 按钮需要比当前普通小按钮略大，保证可点击区域清晰。
- 不展示 `查看知识` 按钮；Phase 2 接入后，查看知识入口改为点击整张知识库卡片。
- 点击卡片内部的 `编辑`、`删除` 按钮时，只触发对应按钮操作，不触发查看知识入口。

### 8.1 状态展示

| 状态     | 展示文字 | 样式       |
| -------- | -------- | ---------- |
| active   | 启用     | 绿色、加粗 |
| disabled | 禁用     | 红色、加粗 |

### 8.2 指标展示

知识库卡片需要展示：

- 文档数量，从 `documents.length` 派生。
- Chunks 数量，从 `documents[].chunks.length` 汇总派生。
- TopK 值。
- 相似度得分阈值。
- 最后更新时间。

字段兜底：

| 字段           | 兜底值         |
| -------------- | -------------- |
| 知识库名称     | `未命名知识库` |
| 描述           | `暂无描述`     |
| 文档数量       | `0`            |
| Chunks 数量    | `0`            |
| TopK           | `0`            |
| 相似度得分阈值 | `0`            |
| 最后更新时间   | `--`           |
| status         | `disabled`     |

## 9. 新建和编辑知识库弹窗

点击 `新建知识库` 按钮后，打开新建知识库弹窗。

点击知识库卡片中的 `编辑` 按钮后，打开编辑知识库弹窗。

新建弹窗和编辑弹窗 UI 与字段保持一致。

### 9.1 可编辑字段

| 字段                | 类型              | 说明                   |
| ------------------- | ----------------- | ---------------------- |
| name                | string            | 知识库名称             |
| description         | string            | 知识库描述             |
| icon                | KnowledgeBaseIcon | 知识库展示图标         |
| topK                | number            | 检索时返回的 TopK 数量 |
| similarityThreshold | number            | 相似度得分阈值         |
| chunkSize           | number            | 分片大小               |
| status              | active / disabled | 是否启用               |

### 9.2 新建默认值

新建弹窗初始字段：

- `name` 为空。
- `description` 为空。
- `icon` 默认值为 `Database`。
- `topK` 使用默认正整数。
- `chunkSize` 使用默认正整数。
- `similarityThreshold` 使用 `0-1` 范围内默认值。
- `status` 默认值为 `active`。

### 9.2.1 图标选择

新建和编辑弹窗需要包含图标选择下拉框：

- 下拉框选项为 `Database`、`BookOpen`、`FileText`、`Folder`、`Archive`、`Brain`、`Bot`、`GraduationCap`、`BriefcaseBusiness`、`Lightbulb`。
- 每个选项需要展示对应图标和名称，便于用户识别。
- 保存后，知识库卡片左上角图标立即同步更新。
- 图标对应颜色随 `icon` 字段同步更新。
- 当前阶段不提供自定义图标或自定义颜色能力。

### 9.3 表单校验

保存前必须校验：

- `name` 必填。
- `topK` 必须为正整数。
- `chunkSize` 必须为正整数。
- `similarityThreshold` 范围为 `0-1`。
- 新建时，名称不能与任意已有知识库重名。
- 编辑时，名称重复校验需要排除当前编辑知识库 id。

校验失败时：

- 不能保存。
- 在保存按钮上方展示一行红色错误提示，说明失败原因。

### 9.4 状态切换

状态字段使用滑动开关实现。

- 开启时表示 `启用`。
- 关闭时表示 `禁用`。

### 9.5 弹窗操作

弹窗底部包含：

- 取消按钮
- 确认按钮

点击取消：

- 关闭弹窗。
- 不保存修改。

点击确认：

- 校验表单。
- 校验通过后写入 Zustand 知识库状态。
- 关闭弹窗。
- 更新对应知识库卡片展示。

## 10. 删除知识库确认框

点击知识库卡片中的 `删除` 按钮后，弹出删除确认框。

### 10.1 关闭方式

以下操作均视为取消删除：

- 点击右上角关闭图标。
- 点击取消按钮。
- 点击弹窗外部遮罩区域。

### 10.2 确认删除

只有点击确认按钮时，才真正删除该知识库。

删除后：

- 当前知识库卡片从页面中移除。
- 统计卡片数量同步更新。
- Zustand 知识库状态同步更新。

## 11. 前端数据兜底要求

### 11.1 列表数据兜底

如果接口返回或初始化数据不是数组，则使用空数组：

```ts
const safeKnowledgeBases = Array.isArray(data) ? data : [];
```

### 11.2 字段兜底

渲染知识库卡片时，应对字段进行兜底：

```ts
const name = typeof item.name === "string" ? item.name : "未命名知识库";
const description =
  typeof item.description === "string" ? item.description : "暂无描述";
const documents = Array.isArray(item.documents) ? item.documents : [];
const documentCount = documents.length;
const chunkCount = documents.reduce(
  (sum, document) =>
    sum + (Array.isArray(document.chunks) ? document.chunks.length : 0),
  0
);
const topK = typeof item.topK === "number" ? item.topK : 0;
const similarityThreshold =
  typeof item.similarityThreshold === "number" ? item.similarityThreshold : 0;
const status =
  item.status === "active" || item.status === "disabled"
    ? item.status
    : "disabled";
const icon = isKnowledgeBaseIcon(item.icon) ? item.icon : "Database";
```

## 12. 空状态与异常状态

### 12.1 知识库为空

当知识库列表为空时，展示空状态。

空状态内容建议：

- 文案：`暂无知识库`
- 辅助文案：`点击新建按钮创建你的第一个知识库`
- 操作按钮：`新建知识库`

### 12.2 搜索无结果

当搜索或筛选后无结果时，展示搜索空状态。

建议文案：

- `没有找到符合条件的知识库`

### 12.3 数据异常

当数据结构异常时：

- 页面不崩溃。
- 使用空数组或字段默认值兜底。
- 可以在控制台输出 warning，方便开发调试。

## 14. Phase 1 验收标准

完成后应满足以下条件：

1. 知识库管理页面能够在管理后台 Main Content 中渲染。
2. 知识库页面顶部包含搜索框、排序下拉框、排序倒置按钮和新建按钮。
3. 搜索框支持输入关键词后按 Enter 搜索。
4. 输入过程中不实时过滤。
5. 清空搜索输入后立即展示全部知识库。
6. 排序下拉框支持 `更新时间` 和 `包含文档数量` 两个选项。
7. 默认按更新时间倒序排序，最新知识库在前。
8. 排序倒置按钮可以反转当前卡片展示顺序。
9. 页面展示知识库总量、启用数量、禁用数量三个统计卡片。
10. 统计卡片数量始终按全部知识库计算，不受搜索和排序影响。
11. 点击统计卡片可以快捷筛选全部、启用或禁用知识库。
12. 再次点击已高亮统计卡片可以取消筛选。
13. 知识库以卡片形式展示。
14. 每个知识库卡片展示名称、描述、图标、文档数量、Chunks 数量、TopK、相似度阈值、更新时间和状态。
15. 文档数量从 `documents.length` 派生。
16. Chunks 数量从 `documents[].chunks.length` 汇总派生。
17. 知识库状态以 `active` 绿色启用、`disabled` 红色禁用展示。
