# RAG 知识库管理模块 Spec

## 1. 模块目标

本模块用于实现 RAG 知识库管理相关的前端页面，主要支持用户查看、筛选、排序、创建、编辑、启用/禁用、删除知识库，并管理知识库下关联的知识文档和知识分片。

当前阶段以前端页面和交互闭环为主，后端接口数据后续统一接入。前端需要对接口返回数据做类型兜底，避免因字段缺失或类型异常导致页面崩溃。

---

## 2. 布局与路由依赖

本模块在管理后台整体布局内渲染。Header、Sidebar、Main Content、侧边栏导航、预留页面和 `/` 到 `/knowledge-bases` 的默认重定向要求，统一参考 `docs/specs/layoutspec.md`。

本 spec 只描述 `/knowledge-bases` 知识库管理页面的业务内容和交互，不重复定义通用布局。

---

## 3. 知识库管理页面

知识库管理页面用于展示和管理已经建立好的知识库。

页面从上到下分为三部分：

1. 筛选操作栏
2. 数据统计卡片区
3. 知识库卡片展示区

---

## 4. 筛选操作栏

筛选操作栏位于页面顶部，包含四个控件：

1. 搜索框
2. 排序下拉框
3. 排序倒置按钮
4. 新建知识库按钮

### 4.1 搜索框

用户可以通过输入关键词搜索知识库。

交互要求：

- 用户输入关键词后，按 `Enter` 键触发搜索。
- 搜索结果只展示符合搜索条件的知识库。
- 搜索范围至少包括：

  - 知识库名称
  - 知识库描述

- 当搜索关键词为空时，展示全部知识库。
- 清空搜索输入后，展示全部知识库。

### 4.2 排序下拉框

下拉框仅用于切换知识库排序方式，不负责状态筛选。

选项包括：

| 选项         | 含义               |
| ------------ | ------------------ |
| 更新时间     | 按最后更新时间排序 |
| 包含文档数量 | 按文档数量排序     |

默认选项为 `更新时间`。

状态筛选由下方统计卡片负责，下拉框与统计卡片不再联动。

### 4.3 排序倒置按钮

点击排序倒置按钮后，当前知识库卡片展示顺序反转。

要求：

- 每次点击切换一次排序方向。
- 可用于反转当前搜索、排序和统计卡片筛选后的展示结果。
- 推荐使用图标按钮表示，例如上下箭头、排序图标等。

### 4.4 新建知识库按钮

新建按钮为蓝底白字。

交互要求：

- 点击后打开 `新建知识库弹窗`。
- 弹窗用于创建新的知识库。
- 当前阶段可以使用前端本地状态模拟创建结果。

---

## 5. 数据统计卡片区

筛选操作栏下方展示三个数据统计卡片：

| 卡片           | 展示内容                        | Icon 颜色 |
| -------------- | ------------------------------- | --------- |
| 知识库总量     | 当前知识库总数量                | 蓝色      |
| 启用知识库数量 | status 为 active 的知识库数量   | 绿色      |
| 禁用知识库数量 | status 为 disabled 的知识库数量 | 红色      |

统计卡片数量始终按全部知识库计算，与搜索关键词、排序方式和当前卡片筛选状态无关。

### 5.1 统计卡片点击筛选

三个统计卡片同时具备快捷筛选能力。

#### 点击“知识库总量”卡片

- 高亮总量卡片。
- 展示全部知识库。
- 不改变排序下拉框当前选项。

#### 点击“启用知识库数量”卡片

- 高亮启用卡片。
- 只展示启用状态知识库。
- 不改变排序下拉框当前选项。

#### 点击“禁用知识库数量”卡片

- 高亮禁用卡片。
- 只展示禁用状态知识库。
- 不改变排序下拉框当前选项。

#### 再次点击已高亮卡片

- 取消当前卡片高亮。
- 展示全部知识库。
- 不改变排序下拉框当前选项。

---

## 6. 知识库展示卡片

知识库默认以卡片形式排列展示。

### 6.1 卡片内容

每个知识库卡片展示以下信息：

| 区域       | 内容                                    |
| ---------- | --------------------------------------- |
| 左上角     | 知识库 Logo / Icon                      |
| 标题区域   | 知识库名称                              |
| 描述区域   | 知识库描述，使用较小字号                |
| 指标区域   | 文档数量、Chunks 数量、TopK、相似度阈值 |
| 时间区域   | 最后更新时间                            |
| 右上角     | 知识库状态                              |
| 底部操作区 | 编辑、查看知识、删除按钮                |

### 6.2 状态展示

知识库状态有两种：

| 状态     | 展示文字 | 样式       |
| -------- | -------- | ---------- |
| active   | 启用     | 绿色、加粗 |
| disabled | 禁用     | 红色、加粗 |

### 6.3 指标展示

知识库卡片需要展示：

- 文档数量
- Chunks 数量
- TopK 值
- 相似度得分阈值
- 最后更新时间

如果字段缺失，需要使用兜底值：

| 字段           | 兜底值     |
| -------------- | ---------- |
| 文档数量       | `0`        |
| Chunks 数量    | `0`        |
| TopK           | `0`        |
| 相似度得分阈值 | `0`        |
| 最后更新时间   | `--`       |
| 描述           | `暂无描述` |

---

## 7. 知识库卡片操作

每个知识库卡片底部包含三个操作按钮：

1. 编辑
2. 查看知识
3. 删除

---

## 8. 编辑知识库弹窗

点击知识库卡片中的 `编辑` 按钮后，打开编辑知识库弹窗。

### 8.1 可编辑字段

编辑弹窗支持修改以下字段：

| 字段           | 类型               | 说明                   |
| -------------- | ------------------ | ---------------------- |
| name           | string             | 知识库名称             |
| description    | string             | 知识库描述             |
| topK           | number             | 检索时返回的 TopK 数量 |
| similarityThreshold | number        | 相似度得分阈值         |
| chunkSize      | number             | 分片大小               |
| status         | active / disabled  | 是否启用               |

表单校验规则：

- `name` 必填。
- `topK` 必须为正整数。
- `chunkSize` 必须为正整数。
- `similarityThreshold` 范围为 `0-1`。
- 名称重复或其他字段不符合规则时不能保存。
- 校验失败时，在保存按钮上方展示一行红色错误提示，说明失败原因。

### 8.2 状态切换

状态字段使用滑动开关实现。

- 开启时表示 `启用`。
- 关闭时表示 `禁用`。

### 8.3 弹窗操作

弹窗底部包含：

- 取消按钮
- 确认按钮

点击取消：

- 关闭弹窗。
- 不保存修改。

点击确认：

- 校验表单。
- 保存修改。
- 关闭弹窗。
- 更新对应知识库卡片展示。

当前阶段可以使用前端本地状态模拟保存。

### 8.4 新建知识库弹窗

点击 `新建知识库` 按钮后，打开新建知识库弹窗。

新建弹窗 UI 与编辑弹窗一致，表单字段与编辑弹窗一致。区别如下：

- 新建弹窗初始字段为空或使用默认值。
- `status` 默认值为 `active`。
- 点击确认后执行与编辑弹窗相同的表单校验。
- 校验通过后，将新的知识库加入当前列表。
- 当前阶段可以使用前端本地状态模拟创建。

---

## 9. 查看知识弹窗

点击知识库卡片中的 `查看知识` 按钮后，打开查看知识弹窗。

该弹窗用于管理当前知识库关联的知识文档。

弹窗分为上下两部分：

1. 文件上传区域
2. 文档列表区域

---

## 10. 文件上传区域

上传区域位于查看知识弹窗上半部分。

交互要求：

- 支持点击上传区域打开本地文件选择器。
- 支持拖拽文件到上传区域。
- 上传成功后，将文件加入下方文档列表。
- 后端服务会自动完成文档分片，并将分片结果同步回前端供分片弹窗展示。
- 当前阶段如果未接入真实后端，可以使用前端 mock 模拟“上传后生成文档与分片”的结果。

建议支持的文件类型：

- PDF
- DOCX
- TXT
- Markdown

### 如果当前阶段不做文件类型校验，可以在代码中预留校验逻辑。

## 11. 文档列表区域

文档列表位于查看知识弹窗下半部分。

表格列包括：

| 列名     | 字段       |
| -------- | ---------- |
| 名称     | name       |
| 大小     | size       |
| 上传时间 | uploadedAt |
| 操作     | actions    |

### 11.1 操作列

每条文档记录包含两个操作按钮：

1. 分片
2. 删除

---

## 12. 删除文档

点击文档列表中的 `删除` 按钮后：

- 删除当前知识库下对应文档。
- 删除文档后，当前知识库卡片展示的文档数量和 Chunks 数量需要同步减少。
- 当前阶段可以直接从前端本地状态中移除该文档，并同步更新对应统计。
- 后续接入接口后，应调用删除文档接口。

---

## 13. 查看分片弹窗

点击文档列表中的 `分片` 按钮后，打开知识分片弹窗。

### 13.1 分片弹窗内容

该弹窗展示当前知识文档下的所有知识分片。

分片内容至少包括：

- 分片序号
- 分片文本内容
- 字符数 / token 数，若数据中存在
- 创建时间，若数据中存在

### 13.2 当前阶段限制

当前阶段知识分片弹窗只读，不支持编辑。

---

## 14. 删除知识库确认框

点击知识库卡片中的 `删除` 按钮后，弹出删除确认框。

### 14.1 关闭方式

以下操作均视为取消删除：

- 点击右上角关闭图标
- 点击取消按钮
- 点击弹窗外部遮罩区域

### 14.2 确认删除

只有点击确认按钮时，才真正删除该知识库。

删除后：

- 当前知识库卡片从页面中移除。
- 统计卡片数量同步更新。
- 当前阶段可以使用前端本地状态模拟删除。

---

## 15. 数据结构建议

数据库的数据结构参考 `DataType.md` 文件和 `prisma/schema.prisma`。

知识库状态统一采用数据库字段值：

| 字段值   | 页面展示 |
| -------- | -------- |
| active   | 启用     |
| disabled | 禁用     |

前端展示数据中允许包含后端服务计算字段：

| 字段          | 含义                                   |
| ------------- | -------------------------------------- |
| documentCount | 当前知识库下文档数量，由后端服务计算   |
| chunkCount    | 当前知识库下 Chunks 数量，由后端服务计算 |

`documentCount` 和 `chunkCount` 不是数据库表中的直接字段。后端接入后由服务层根据文档和分片数据计算得到；当前前端 mock 阶段可以直接在本地数据中维护。

前端展示模型建议如下：

```ts
type KnowledgeBaseViewModel = {
  id: string;
  name: string;
  description: string;
  icon?: string;
  documentCount: number;
  chunkCount: number;
  topK: number;
  chunkSize: number;
  similarityThreshold: number;
  status: "active" | "disabled";
  updatedAt: string;
  documents: KnowledgeDocumentViewModel[];
};
```

## 16. 前端数据兜底要求

由于当前阶段后端返回数据结构尚未完全确定，前端需要做数据兜底处理。

### 16.1 列表数据兜底

如果接口返回的知识库列表不是数组，则使用空数组：

```ts
const safeKnowledgeBases = Array.isArray(data) ? data : [];
```

### 16.2 字段兜底

渲染知识库卡片时，应对字段进行兜底：

```ts
const name = typeof item.name === "string" ? item.name : "未命名知识库";
const description =
  typeof item.description === "string" ? item.description : "暂无描述";
const documentCount =
  typeof item.documentCount === "number" ? item.documentCount : 0;
const chunkCount = typeof item.chunkCount === "number" ? item.chunkCount : 0;
const topK = typeof item.topK === "number" ? item.topK : 0;
const similarityThreshold =
  typeof item.similarityThreshold === "number" ? item.similarityThreshold : 0;
const status =
  item.status === "active" || item.status === "disabled"
    ? item.status
    : "disabled";
```

### 16.3 子列表兜底

文档列表和分片列表也需要做数组兜底：

```ts
const safeDocuments = Array.isArray(item.documents) ? item.documents : [];
const safeChunks = Array.isArray(document.chunks) ? document.chunks : [];
```

## 17. 空状态与异常状态

### 17.1 知识库为空

当知识库列表为空时，展示空状态。

空状态内容建议：

- 文案：`暂无知识库`
- 辅助文案：`点击新建按钮创建你的第一个知识库`
- 操作按钮：`新建知识库`

### 17.2 搜索无结果

当搜索或筛选后无结果时，展示搜索空状态。

建议文案：

- `没有找到符合条件的知识库`

### 17.3 数据异常

当接口返回数据结构异常时：

- 页面不崩溃。
- 使用空数组或字段默认值兜底。
- 可以在控制台输出 warning，方便开发调试。

## 20. 验收标准

完成后应满足以下条件：

1. 知识库管理页面能够在 `docs/specs/layoutspec.md` 定义的 Main Content 中渲染。
2. 知识库页面顶部包含搜索框、排序下拉框、排序倒置按钮和新建按钮。
3. 搜索框支持输入关键词后按 Enter 搜索，清空搜索输入后展示全部知识库。
4. 排序下拉框支持更新时间、包含文档数量两个选项，默认按更新时间排序。
5. 排序倒置按钮可以反转当前卡片展示顺序。
6. 页面展示知识库总量、启用数量、禁用数量三个统计卡片。
7. 统计卡片数量始终按全部知识库计算，不受搜索和排序影响。
8. 点击统计卡片可以快速筛选全部、启用或禁用知识库，但不改变排序下拉框状态。
9. 知识库以卡片形式展示。
10. 每个知识库卡片展示名称、描述、图标、文档数量、chunks 数量、TopK、相似度阈值、更新时间和状态。
11. 知识库状态以 `active` 绿色启用、`disabled` 红色禁用展示。
12. 点击新建按钮可以打开新建知识库弹窗，新建弹窗 UI 和字段与编辑弹窗一致，默认状态为 `active`。
13. 点击编辑按钮可以打开编辑知识库弹窗。
14. 新建和编辑弹窗可以修改名称、描述、TopK、相似度阈值、chunkSize 和启用状态。
15. 表单校验失败时不能保存，并在保存按钮上方展示红色错误提示。
16. 点击查看知识按钮可以打开知识文档管理弹窗。
17. 查看知识弹窗包含上传区域和文档列表。
18. 上传文件后，文档列表新增对应文档，并能展示后端同步回来的分片结果；mock 阶段可用本地数据模拟。
19. 文档列表包含名称、大小、上传时间和操作列。
20. 点击文档删除按钮可以删除对应文档，并同步减少知识库卡片的文档数量和 Chunks 数量。
21. 点击文档分片按钮可以打开分片查看弹窗。
22. 分片查看弹窗只读展示分片内容。
23. 点击知识库删除按钮会弹出确认框。
24. 只有点击确认按钮才会真正删除知识库。
25. 关闭图标、取消按钮、点击弹窗外部均不会删除知识库。
26. 当接口数据不是数组时，页面使用空数组兜底。
27. 当字段缺失或类型异常时，页面使用默认值兜底，不发生崩溃。
