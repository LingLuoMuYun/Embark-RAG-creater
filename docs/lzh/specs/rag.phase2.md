# RAG 知识文档与分片管理模块 Phase 2 Spec

## 1. 阶段目标

Phase 2 在 Phase 1 的知识库管理能力基础上，实现单个知识库下的知识文档和知识分片管理。

Phase 2 范围包括：

- 知识库卡片支持整卡点击打开查看知识。
- 查看当前知识库的知识文档管理弹窗。
- 文件上传区域。
- 上传 mock 逻辑。
- 文档列表展示。
- 删除文档。
- 查看分片弹窗。
- 分片只读展示。
- 文档和分片数据兜底。

Phase 2 不重复实现知识库列表、新建、编辑、删除能力，这些能力由 `docs/lzh/specs/rag.phase1.md` 定义。

---

## 2. 依赖前提

Phase 2 依赖 Phase 1 已完成：

- `/knowledge-bases` 知识库管理页面。
- 知识库卡片列表。
- Zustand 全局知识库状态。
- `src/features/knowledge-bases/mock-data.ts` mock 数据。
- 主列表知识库数据只保存轻量字段。
- 主列表知识库数据使用 `documentCount` 和 `chunkCount` 展示统计数量。
- `src/store/slices/knowledge-base-slice.ts` 已预留 `selectedId`、`selected`、`selectedDocs`、`selectedChunks`。

Phase 2 必须继续遵循 Phase 1 的状态边界：

- `items` 只保存知识库轻量列表。
- 不把 `documents` 或 `chunks` 明细写进 `items`。
- 当前查看的知识库详情写入 `selected`。
- 当前查看知识库的文档列表写入 `selectedDocs`。
- 当前查看的分片列表写入 `selectedChunks`。
- 文档和分片详情只围绕当前选中的知识库维护。

---

## 3. 知识库卡片查看知识入口

Phase 2 不再新增 `查看知识` 按钮。查看知识入口改为点击整张知识库卡片。

每个知识库卡片底部操作按钮保持为：

1. 编辑
2. 删除

点击知识库卡片主体后：

- 设置当前知识库为选中知识库。
- 打开当前知识库的知识文档管理弹窗。
- 模拟加载当前知识库详情。
- 后端不可用或详情接口未接入时，使用 mock 详情兜底。
- 当前阶段初次打开某个知识库时，文档列表为空。

交互限制：

- 点击卡片内部的 `编辑` 按钮只打开编辑弹窗。
- 点击卡片内部的 `删除` 按钮只打开删除确认框。
- `编辑`、`删除` 按钮需要阻止事件冒泡，避免误触发查看知识弹窗。
- 卡片 hover 样式需要提示整卡可点击，例如轻微放大、阴影增强或边框颜色变化。

---

## 4. 查看知识弹窗

查看知识弹窗用于管理当前知识库关联的知识文档。

弹窗分为上下两部分：

1. 文件上传区域
2. 文档列表区域

弹窗标题应能体现当前知识库，例如：

```text
查看知识 - <知识库名称>
```

弹窗中的文档列表来自 `selectedDocs`，不是来自知识库列表项中的 `documents`。

---

## 5. 详情加载规则

Phase 2 当前阶段仍以前端 mock 为主，不要求真实接口联调。

点击知识库卡片主体时需要模拟详情请求流程：

1. 设置 `selectedId` 为当前知识库 id。
2. 将当前知识库基础信息写入 `selected`。
3. 模拟加载当前知识库文档列表。
4. 初次打开某个知识库时，`selectedDocs` 为空数组。
5. `selectedChunks` 初始为空数组。

后续接入真实接口时，可以替换为：

```text
GET /api/knowledge-bases/:id
GET /api/knowledge-bases/:id/documents/:documentId/chunks
```

当前阶段如果保留请求入口，接口失败时必须回退到 mock 详情，不影响弹窗打开和后续上传 mock。

---

## 6. 文件上传区域

上传区域位于查看知识弹窗上半部分。

交互要求：

- 支持点击上传区域打开本地文件选择器。
- 支持拖拽文件到上传区域。
- 不支持一次选择多个文件。
- 当前阶段不上传到真实服务器。
- 当前阶段不手动实现真实文档解析。
- 当前阶段使用前端 mock 模拟“上传后生成文档与分片”的结果。

支持的文件类型：

- PDF
- DOCX
- TXT
- Markdown

文件限制：

- 文件大小上限为 20MB。
- 不支持的文件类型需要直接拦截，并给出用户可见错误提示。
- 超过大小限制的文件需要直接拦截，并给出用户可见错误提示。
- 同一知识库下不允许上传同名文件。
- 重名校验只针对当前 `selectedDocs`。

---

## 7. 上传 mock 规则

当前阶段上传 mock 按以下规则执行：

1. 用户每上传 1 个文件，生成 1 条文档记录。
2. 文档 `name` 使用文件真实名称。
3. 文档 `size` 使用文件真实 size。
4. 文档 `uploadedAt` 使用当前时间。
5. 文档 `id` 使用前端生成的唯一 id。
6. 每个上传文件随机 mock 生成 2-3 条 chunks。
7. chunk `content` 使用固定模板，例如：

```text
这是从 <文件名> 生成的模拟知识分片 <序号>。
```

8. chunk `charCount` 根据 `content.length` 计算。
9. chunk `tokenCount` 当前阶段可以使用简单估算值或省略。
10. chunk `createdAt` 使用当前时间。

上传成功后：

- 新文档写入当前知识库的 `selectedDocs`。
- 生成的 chunks 与该文档建立关联。
- 如果用户立即查看该文档分片，则将该文档对应 chunks 写入 `selectedChunks`。
- 同步更新 `items` 中当前知识库的 `documentCount`。
- 同步更新 `items` 中当前知识库的 `chunkCount`。
- 不把 documents/chunks 明细写进 `items`。

---

## 8. 文档列表区域

文档列表位于查看知识弹窗下半部分。

表格列包括：

| 列名 | 字段 |
| --- | --- |
| 名称 | name |
| 大小 | size |
| 上传时间 | uploadedAt |
| 操作 | actions |

### 8.1 操作列

每条文档记录包含两个操作按钮：

1. 分片
2. 删除

点击 `分片` 按钮时：

- 打开分片查看弹窗。
- 将当前文档对应的 chunks 写入 `selectedChunks`。

点击 `删除` 按钮时：

- 打开删除文档确认框。
- 不直接删除文档。

---

## 9. 删除文档

点击文档列表中的 `删除` 按钮后，弹出删除文档确认框。

确认框要求：

- 标题为 `确认删除文档`。
- 正文需要包含当前 RAG 知识库名称。
- 正文需要包含将被删除的文档名称。
- 正文需要说明删除后该文档和关联 mock 分片会从当前知识库中移除。
- 按钮包含 `取消` 和 `确认删除`。
- `确认删除` 按钮使用红色危险样式。

点击 `取消` 或关闭弹窗：

- 不删除文档。
- 不更新 `selectedDocs`。
- 不更新 `selectedChunks`。
- 不更新 `items`。

点击 `确认删除`：

- 从 `selectedDocs` 中移除当前文档。
- 如果当前 `selectedChunks` 属于被删除文档，则清空 `selectedChunks`。
- 同步更新 `items` 中当前知识库的 `documentCount`。
- 同步更新 `items` 中当前知识库的 `chunkCount`。
- 不把 documents/chunks 明细写进 `items`。

后续接入接口后，应调用删除文档接口。

---

## 10. 查看分片弹窗

点击文档列表中的 `分片` 按钮后，打开知识分片弹窗。

该弹窗展示当前知识文档下的所有知识分片。

### 10.1 分片弹窗内容

分片内容至少包括：

- 分片序号。
- 分片文本内容。
- 字符数，若数据中存在。
- token 数，若数据中存在。
- 创建时间，若数据中存在。

### 10.2 只读限制

Phase 2 知识分片弹窗只读，不支持：

- 编辑分片。
- 删除分片。
- 新增分片。
- 重新切分。

---

## 11. 数据结构要求

Phase 2 沿用 Phase 1 已落地的类型方向，文档和分片数据不进入主列表 `items`。

文档数据建议结构：

```ts
type RagDoc = {
  id: string;
  name: string;
  size: number;
  uploadedAt: string;
};
```

分片数据建议结构：

```ts
type RagChunk = {
  id: string;
  documentId: string;
  content: string;
  tokenCount?: number;
  charCount?: number;
  createdAt?: string;
};
```

说明：

- `selectedDocs` 保存当前知识库文档列表。
- `selectedChunks` 保存当前正在查看的文档分片列表。
- chunk 需要能通过 `documentId` 关联到所属文档。
- 主列表 `items` 只保存 `documentCount` 和 `chunkCount` 统计字段。

---

## 12. 数据兜底要求

### 12.1 文档列表兜底

如果当前知识库文档列表不是数组，则使用空数组：

```ts
const safeDocuments = Array.isArray(selectedDocs) ? selectedDocs : [];
```

### 12.2 分片列表兜底

如果当前分片列表不是数组，则使用空数组：

```ts
const safeChunks = Array.isArray(selectedChunks) ? selectedChunks : [];
```

### 12.3 字段兜底

文档字段兜底：

| 字段 | 兜底值 |
| --- | --- |
| name | `未命名文档` |
| size | `0` |
| uploadedAt | `--` |

分片字段兜底：

| 字段 | 兜底值 |
| --- | --- |
| content | `暂无内容` |
| charCount | 不展示或 `0` |
| tokenCount | 不展示或 `0` |
| createdAt | `--` |

---

## 13. 空状态

### 13.1 文档为空

当当前知识库没有文档时，文档列表区域展示空状态。

建议文案：

- `暂无知识文档`
- `上传文件后将在这里展示文档`

### 13.2 分片为空

当当前文档没有分片时，分片弹窗展示空状态。

建议文案：

- `暂无知识分片`

---

## 14. Phase 2 暂不实现

Phase 2 暂不实现：

- 真实文件上传到服务器。
- 真实文档解析。
- 真实知识分片生成。
- 一次选择多个文件上传。
- 分片编辑。
- 分片删除。
- 分片重新切分。
- 权限控制。
- 多租户隔离。

## 15. Phase 2 验收标准

完成后应满足以下条件：

1. 知识库卡片底部不包含 `查看知识` 按钮。
2. 点击知识库卡片主体可以打开当前知识库的知识文档管理弹窗。
3. 点击知识库卡片主体时设置当前知识库为选中知识库。
4. 点击知识库卡片内的 `编辑` 或 `删除` 按钮不会打开查看知识弹窗。
5. 初次打开某个知识库时，文档列表为空。
6. 查看知识弹窗包含文件上传区域和文档列表区域。
7. 上传区域支持点击选择文件。
8. 上传区域支持拖拽文件。
9. 不支持一次选择多个文件。
10. 不支持的文件类型会被拦截并展示错误提示。
11. 超过 20MB 的文件会被拦截并展示错误提示。
12. 同一知识库下不允许上传同名文件。
13. 每上传 1 个文件，生成 1 条文档记录。
14. 文档名称使用文件真实名称。
15. 文档大小使用文件真实 size。
16. 文档上传时间使用当前时间。
17. 每个上传文件随机 mock 生成 2-3 条知识分片。
18. 上传后文档列表新增对应文档。
19. 上传后知识库卡片文档数量同步更新。
20. 上传后知识库卡片 Chunks 数量同步更新。
21. 文档和分片明细不会写入主列表 `items`。
22. 文档列表包含名称、大小、上传时间和操作列。
23. 点击文档删除按钮会打开删除确认框。
24. 删除确认框正文包含当前 RAG 知识库名称和待删除文档名称。
25. 删除确认框包含 `取消` 和红色 `确认删除` 按钮。
26. 点击取消不会删除文档。
27. 点击确认删除后，从 `selectedDocs` 中删除对应文档。
28. 删除文档后知识库卡片文档数量同步减少。
29. 删除文档后知识库卡片 Chunks 数量同步减少。
30. 点击文档分片按钮可以打开分片查看弹窗。
31. 分片查看弹窗只读展示分片内容。
32. 文档列表异常时使用空数组兜底。
33. 分片列表异常时使用空数组兜底。
34. 文档或分片字段缺失时使用默认值兜底，不发生页面崩溃。
