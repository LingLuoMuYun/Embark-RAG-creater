# 管理后台整体布局 Spec

## 1. 模块目标

本规范用于定义项目管理后台的整体布局、基础导航和页面承载方式。业务模块页面，例如知识库管理、知识文档和专家 Agent，应在该布局内渲染，避免各业务 spec 重复描述通用布局。

当前阶段重点实现：

- 上边栏 Header
- 左侧边栏 Sidebar
- 主体内容 Main Content
- 后台基础路由和默认重定向
- 预留用户信息区域

---

## 2. 页面整体布局

页面采用三段式管理后台布局：

```text
上边栏 Header
左侧边栏 Sidebar
主体内容 Main Content
```

### 2.1 上边栏 Header

上边栏展示当前功能页面名称。

要求：

- 左侧展示当前页面标题。
- 进入知识库页面时，标题显示为 `知识库管理`。
- 右侧预留用户信息区域，例如用户头像、用户名、设置入口等。
- 当前阶段用户信息区域可以仅保留占位，不要求实现完整用户系统或真实头像数据。

### 2.2 左侧边栏 Sidebar

左侧边栏支持收缩和展开。

侧边栏包含以下导航项：

- 知识库
- 知识文档
- 专家 Agent

导航要求：

- 点击 `知识库` 时，路由跳转到 `/knowledge-bases`。
- 点击 `知识文档` 时，路由跳转到 `/documents`。
- 点击 `专家 Agent` 时，路由跳转到 `/agents`。
- 当前路由对应的导航项应有明确选中状态。

### 2.3 主体内容 Main Content

主体内容根据当前路由展示对应页面内容。

点击侧边栏 `知识库` 导航项时：

- 路由跳转至知识库管理页面。
- 上边栏标题变更为 `知识库管理`。
- 主体区域展示知识库管理页面内容。

---

## 3. 路由设计

建议路由如下：

| 页面       | 路由               | 说明               |
| ---------- | ------------------ | ------------------ |
| 默认入口   | `/`                | 重定向到 `/knowledge-bases` |
| 知识库管理 | `/knowledge-bases` | 展示知识库卡片列表 |
| 知识文档   | `/documents`       | 预留页面           |
| 专家 Agent | `/agents`          | 预留页面           |

如果项目已有路由规范，应优先遵循现有项目结构。

根路径 `/` 应重定向到 `/knowledge-bases`，进入系统后默认展示知识库管理页面。

---

## 4. 预留页面要求

当前阶段重点实现知识库管理页面。`/documents` 和 `/agents` 可以先作为预留页面。

预留页面至少应满足：

- 在管理后台布局内展示。
- Header 标题与当前页面一致。
- 主体区域展示简洁的占位内容。
- 不要求实现真实业务数据、表单或接口联调。

---

## 5. 当前阶段实现范围

当前阶段需要实现：

- 管理后台整体布局
- 上边栏标题展示
- 右侧用户信息占位区域
- 可收缩左侧边栏
- 侧边栏导航项和选中状态
- `/` 到 `/knowledge-bases` 的默认重定向
- `/knowledge-bases`、`/documents`、`/agents` 路由入口
- 业务页面在 Main Content 中渲染

---

## 6. 当前阶段暂不实现

当前阶段暂不要求实现：

- 真实用户系统
- 真实头像数据
- 权限控制
- 多租户隔离
- 复杂面包屑
- 多级菜单

以上能力可以在后续版本中逐步接入。

---

## 7. 验收标准

完成后应满足以下条件：

1. 页面采用上边栏、左侧边栏、主体内容的管理后台布局。
2. 访问 `/` 后，能够重定向到 `/knowledge-bases`。
3. 点击侧边栏 `知识库` 后，能够进入知识库管理页面。
4. 点击侧边栏 `知识文档` 后，能够进入 `/documents` 预留页面。
5. 点击侧边栏 `专家 Agent` 后，能够进入 `/agents` 预留页面。
6. 上边栏标题能够根据当前路由显示对应页面标题。
7. 进入 `/knowledge-bases` 时，上边栏标题显示 `知识库管理`。
8. 左侧边栏支持收缩和展开。
9. 当前路由对应的侧边栏导航项有明确选中状态。
10. 右侧用户信息区域保留占位，不要求接入真实用户系统。
