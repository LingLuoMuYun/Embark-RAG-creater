# 知识闭环与数据分析接口对接文档

对接对象：

- **成员 B（轻量知识检索与组织）**：检索完成后记录命中知识和未命中问题。
- **成员 F（Agent 问答消费与引用）**：Agent 问答完成后记录本次使用到的知识引用。
- **成员 A（知识生命周期管理）**：查看热门知识和知识缺口，用于知识维护。
- **成员 E（专家 Agent 配置与生成）**：查看 Dashboard 总览和 Agent 相关统计。

## 1. 职责边界

成员 G 负责统计和闭环，不负责执行检索，也不负责生成回答。

上游模块只需要在完成检索或问答后，把本次问题、检索范围、命中上下文和引用结果提交给统计接口。统计模块会根据这些数据生成：

- 热门知识排行
- 知识缺口列表
- Dashboard 总览数据
- 近期知识生产热力图

## 2. 对接流程

推荐流程：

```text
用户提问
  ↓
成员 B / 成员 F 调用 POST /api/rag/retrieve
  ↓
拿到 contexts 和 references
  ↓
调用 POST /api/usage/logs 记录本次使用日志
  ↓
成员 G Dashboard 自动统计热门知识和知识缺口
```

## 3. 数据结构

### 3.1 UsageLog

```ts
type UsageLog = {
  id: string;
  source: "rag_retrieve";
  query: string;
  mode: "fast" | "balanced" | "detailed";
  scope: RagRetrieveRequest["scope"];
  hitCount: number;
  noHit: boolean;
  createdAt: string;
  references: UsageReference[];
};
```

### 3.2 UsageReference

字段与 B 的 `POST /api/rag/retrieve` 返回中的 `RagReference` 保持一致。

```ts
type UsageReference = {
  knowledgeBaseId: string;
  knowledgeId: string;
  chunkId: string;
  title: string;
  chunkType: "text" | "wiki" | "summary" | "qa";
};
```

## 4. 接口

所有接口统一返回 `{ success: true, data: ... }`，失败返回 `{ success: false, error: { code, message } }`。

### POST /api/usage/logs

记录一次 RAG 检索或 Agent 问答使用日志。

调用时机：

- 成员 B：`POST /api/rag/retrieve` 返回后调用。
- 成员 F：Agent 问答拿到引用知识后调用。

请求参数：

```ts
type CreateUsageLogRequest = {
  query: string;
  scope: {
    knowledgeBaseIds: string[];
    knowledgeIds?: string[];
    categoryIds?: string[];
    tagIds?: string[];
    chunkTypes?: Array<"text" | "wiki" | "summary" | "qa">;
  };
  mode?: "fast" | "balanced" | "detailed";
  contexts?: RagContext[];
  references?: UsageReference[];
};
```

请求示例：

```json
{
  "query": "如何给成员配置管理员权限？",
  "scope": {
    "knowledgeBaseIds": ["kb_001"],
    "categoryIds": ["cat_permission"],
    "tagIds": ["tag_admin"],
    "chunkTypes": ["text", "wiki"]
  },
  "mode": "balanced",
  "contexts": [
    {
      "id": "ctx_001",
      "knowledgeBaseId": "kb_001",
      "knowledgeId": "wiki_001",
      "chunkId": "chunk_wiki_001",
      "title": "账号权限配置说明",
      "content": "管理员可以在系统设置中进入权限管理页面...",
      "chunkType": "wiki",
      "score": 0.86
    }
  ],
  "references": [
    {
      "knowledgeBaseId": "kb_001",
      "knowledgeId": "wiki_001",
      "chunkId": "chunk_wiki_001",
      "title": "账号权限配置说明",
      "chunkType": "wiki"
    }
  ]
}
```

返回示例：

```json
{
  "success": true,
  "data": {
    "id": "clx_usage_001",
    "query": "如何给成员配置管理员权限？",
    "hitCount": 1,
    "noHit": false,
    "references": [
      {
        "knowledgeBaseId": "kb_001",
        "knowledgeId": "wiki_001",
        "chunkId": "chunk_wiki_001",
        "title": "账号权限配置说明",
        "chunkType": "wiki"
      }
    ],
    "createdAt": "2026-05-30T15:00:00.000Z"
  },
  "message": "检索使用日志记录成功"
}
```

### 未命中记录方式

如果本次检索没有命中知识，传空数组即可：

```json
{
  "query": "如何重置二级密码？",
  "scope": {
    "knowledgeBaseIds": ["kb_001"]
  },
  "mode": "balanced",
  "contexts": [],
  "references": []
}
```

统计模块会自动记录为知识缺口。

## 5. 统计查询接口

### GET /api/analytics/overview

获取 Dashboard 总览数据。

返回内容包括：

- 文档数量
- 已解析文档数量
- 文档分段数量
- 知识库数量
- 知识文档数量
- 知识片段数量
- Agent 数量
- 检索日志数量
- 知识缺口数量
- 最近新增文档
- 近 30 天文档上传热力图
- 热门知识排行
- 知识缺口列表

### GET /api/analytics/hot-knowledge

获取热门知识排行。

查询参数：

```text
limit=10
```

返回示例：

```json
{
  "success": true,
  "data": [
    {
      "knowledgeBaseId": "kb_001",
      "knowledgeId": "wiki_001",
      "title": "账号权限配置说明",
      "hitCount": 12
    }
  ],
  "message": "热门知识排行获取成功"
}
```

### GET /api/analytics/knowledge-gaps

获取知识缺口列表。

查询参数：

```text
limit=10
```

返回示例：

```json
{
  "success": true,
  "data": [
    {
      "query": "如何重置二级密码？",
      "count": 3
    }
  ],
  "message": "知识缺口获取成功"
}
```

## 6. 成员 B / F 推荐调用方式

```ts
const retrieveResult = await fetch("/api/rag/retrieve", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    query,
    scope,
    mode: "balanced",
  }),
}).then((res) => res.json());

await fetch("/api/usage/logs", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    query,
    scope,
    mode: "balanced",
    contexts: retrieveResult.data.contexts,
    references: retrieveResult.data.references,
  }),
});
```

## 7. 注意事项

- `query` 必填，用于统计知识缺口。
- `scope.knowledgeBaseIds` 必填，和 B 的 RAG 检索接口保持一致。
- 如果 `references` 不为空，统计模块会按 `knowledgeId` 和 `title` 汇总热门知识。
- 如果 `contexts` 和 `references` 都为空，统计模块会将本次问题记为知识缺口。
- 如果调用方只传 `contexts`，没有传 `references`，统计模块会从 `contexts` 中提取引用信息。
