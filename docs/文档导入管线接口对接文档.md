# 文档导入管线接口对接文档

对接对象：

- **成员 D（AI 知识提炼）**：获取已解析的文档文本和分段数据，作为 AI 提炼的输入
- **成员 G（数据分析）**：获取文档来源统计数据

## 1. 职责边界

文档导入管线负责文件的上传、解析和文本切分。下游同学只需要拿文本数据，不需要关心文件是怎么解析的。

## 2. 数据模型

```ts
// 文档记录
type DocumentSource = {
  id: string;
  originalName: string;       // 原始文件名
  fileType: string;            // txt | md | csv | xlsx | docx | pdf
  fileSize: number;            // 字节
  status: "uploading" | "uploaded" | "parsing" | "parsed" | "failed";
  content: string | null;      // 解析后的全文，仅 parsed 状态有值
  errorMessage: string | null; // 失败原因
  chunkCount: number;          // 分段数
  createdAt: string;
};

// 文本分段
type DocumentChunk = {
  id: string;
  documentSourceId: string;
  chunkIndex: number;   // 序号，从 0 开始
  content: string;      // 分段文本，可直接喂给 AI
  charStart: number;    // 在原文中的起始位置
  charEnd: number;      // 在原文中的结束位置
};
```

### 状态说明

| 状态 | 含义 |
|------|------|
| `uploading` | 上传中 |
| `uploaded` | 已保存，等待解析 |
| `parsing` | 解析中 |
| `parsed` | 已解析，content 和 chunks 可用 |
| `failed` | 解析失败，可重新解析 |

### 切分说明

- 按段落边界合并至 2000 字符，相邻段 200 字符重叠
- 支持的格式：txt、md、csv、xlsx、docx、pdf（编码自动处理）
- 上传限制：单文件 10MB，支持多文件

## 3. 接口

所有接口统一返回 `{ success: true, data: ... }`，失败返回 `{ success: false, error: { code, message } }`。

### GET /api/documents?status=parsed

文档列表，按时间倒序、支持分页。

```json
{
  "data": {
    "items": [{ "id": "...", "originalName": "技术方案.pdf", "status": "parsed", "chunkCount": 3, ... }],
    "total": 42, "page": 1, "pageSize": 20
  }
}
```

### GET /api/documents/:id

文档详情，含全文 content。

### GET /api/documents/:id/chunks（推荐）

获取分段列表。**AI 提炼同学的主入口**，逐段调用 AI 即可。

```json
{
  "data": [
    { "id": "...", "chunkIndex": 0, "content": "分段文本...", "charStart": 0, "charEnd": 1850 },
    { "chunkIndex": 1, "content": "...", "charStart": 1650, "charEnd": 3800 }
  ]
}
```

### POST /api/documents/:id/parse

触发解析。上传后文档状态为 `uploaded`，需调用此接口。解析成功后返回 content 和 chunkCount。

### POST /api/documents（上传）

`multipart/form-data`，字段 `file`。

### DELETE /api/documents/:id

删除文档，级联删除 chunks 和文件。

## 4. 成员 D 推荐调用方式

```ts
// 1. 获取已解析的文档
const { data } = await fetch("/api/documents?status=parsed").then(r => r.json());

for (const doc of data.items) {
  // 2. 获取分段
  const { data: chunks } = await fetch(`/api/documents/${doc.id}/chunks`).then(r => r.json());

  // 3. 逐段 AI 提炼
  for (const chunk of chunks) {
    const result = await yourAiExtract(chunk.content);
    // 4. 存入 KnowledgeChunk（参考 docs/知识入库字段建议...md）
  }
}
```

## 5. 注意事项

- 只使用 `status === "parsed"` 的文档
- chunks 已按 `chunkIndex` 排序，内容为 UTF-8
- 文档删除后 chunks 级联删除
- 所有接口输入均经过 Zod 校验
