# RAG 检索模块对接字段说明

这份文档只约定 RAG 检索模块与其他同学之间的字段对接。

对接对象只有两类：

- 知识入库同学：负责把知识保存成稳定、可读取、可筛选的知识片段。
- Agent 调用同学：负责传入用户问题和检索范围，接收可交给大模型参考的知识素材。

## 1. 职责边界

入库同学负责保存知识本身，包括普通文档片段、手动知识、LLM Wiki 提炼结果等。

RAG 检索模块负责根据已入库的知识内容自行完成检索准备和检索处理。入库同学不需要在业务表里维护向量字段，也不需要关心检索模块内部使用哪种模型或索引方式。

也就是说：

- 入库侧保证 `title`、`content`、`summary`、分类、标签、状态等字段可靠。
- RAG 侧根据这些字段自行决定如何建立和更新检索索引。
- Agent 侧只需要传 `query` 和 `scope`，不需要传检索参数。

## 2. 给入库同学的字段要求

检索模块主要消费的是“可检索知识片段”。如果第一版还没有复杂分块，一条知识也可以先保存成一个 chunk，`chunkIndex` 设为 `0`。

### 2.1 KnowledgeChunk

```ts
type KnowledgeChunk = {
  /** chunk 唯一 ID，用于定位一段具体知识内容。 */
  id: string;

  /** 所属知识库 ID，用于按知识库范围检索。 */
  knowledgeBaseId: string;

  /** 所属知识 ID，可以对应一篇文档、一条手工知识或一篇 Wiki。 */
  knowledgeId: string;

  /** 知识标题，用于结果展示、引用说明和辅助匹配。 */
  title: string;

  /** chunk 正文，必须是可以直接给大模型阅读的文本。 */
  content: string;

  /** 摘要，可用于保存知识或 Wiki 的简短概括；没有可以不传。 */
  summary?: string;

  /** 分类 ID，用于按分类缩小检索范围。 */
  categoryId?: string;

  /** 标签 ID 列表，用于按标签缩小检索范围。 */
  tagIds?: string[];

  /** 知识状态；检索模块默认只使用 available 的内容。 */
  status: "pending" | "available" | "disabled";

  /** 知识来源类型，用于区分手动录入、文件导入、Wiki 提炼等来源。 */
  sourceType: "manual" | "file" | "wiki" | "import";

  /** chunk 类型，用于区分普通文本、Wiki 内容、摘要、问答类知识。 */
  chunkType: "text" | "wiki" | "summary" | "qa";

  /** 当前 chunk 在所属知识中的顺序，从 0 开始。 */
  chunkIndex: number;

  /** 父 chunk ID；如果有父子片段关系，可以用来挂载更完整的上下文。 */
  parentChunkId?: string;

  /** 前一个 chunk ID；如果同一知识被切成多段，可以保存前后关系。 */
  prevChunkId?: string;

  /** 后一个 chunk ID；如果同一知识被切成多段，可以保存前后关系。 */
  nextChunkId?: string;

  /** Wiki 或摘要内容引用的原始 chunk ID，用于后续做来源追踪。 */
  sourceChunkIds?: string[];

  /** 扩展信息，例如文件名、页码、章节路径、原始链接等。 */
  metadata?: Record<string, unknown>;

  /** 创建时间，建议使用 ISO 时间字符串。 */
  createdAt: string;

  /** 更新时间，内容、状态、分类、标签变化时需要同步更新。 */
  updatedAt: string;
};
```

### 2.2 LLM Wiki 入库示例

LLM Wiki 提炼出的内容也建议按 `KnowledgeChunk` 保存，不需要给检索模块单独设计另一套结构。

```json
{
  "id": "chunk_wiki_001",
  "knowledgeBaseId": "kb_001",
  "knowledgeId": "wiki_001",
  "title": "账号权限配置说明",
  "content": "管理员可以在系统设置中进入权限管理页面，为成员分配不同角色...",
  "summary": "说明账号权限和角色配置方式",
  "categoryId": "cat_permission",
  "tagIds": ["tag_admin", "tag_permission"],
  "status": "available",
  "sourceType": "wiki",
  "chunkType": "wiki",
  "chunkIndex": 0,
  "sourceChunkIds": ["chunk_doc_001", "chunk_doc_002"],
  "metadata": {
    "sectionPath": "账号管理/权限配置",
    "sourceKnowledgeIds": ["doc_001", "doc_002"]
  },
  "createdAt": "2026-05-29T10:00:00.000Z",
  "updatedAt": "2026-05-29T10:00:00.000Z"
}
```

### 2.3 入库注意事项

- `pending` 表示知识还未确认或未完成处理，检索模块默认不使用。
- `available` 表示知识可用于检索和回答。
- `disabled` 表示知识不可用于检索。
- `content` 不建议保存为空；如果是 Wiki，正文应保存提炼后的完整可读内容。
- 如果能保存 `prevChunkId`、`nextChunkId`、`parentChunkId`、`sourceChunkIds`，后续更方便做上下文组织和来源追踪。

## 3. 给 Agent 调用同学的接口约定

Agent 或问答模块在需要知识素材时，请调用 RAG 检索接口。

```http
POST /api/rag/retrieve
```

### 3.1 请求参数

```ts
type RagRetrieveRequest = {
  /** 用户原始问题。 */
  query: string;

  /** 本次允许检索的范围。 */
  scope: {
    /** 允许检索的知识库 ID 列表，必填。 */
    knowledgeBaseIds: string[];

    /** 限定具体知识 ID；不传表示不限定具体知识。 */
    knowledgeIds?: string[];

    /** 限定分类 ID；不传表示不限定分类。 */
    categoryIds?: string[];

    /** 限定标签 ID；不传表示不限定标签。 */
    tagIds?: string[];

    /** 限定知识片段类型；不传表示不限定类型。 */
    chunkTypes?: Array<"text" | "wiki" | "summary" | "qa">;
  };

  /** 返回素材的详细程度偏好；不传默认 balanced。 */
  mode?: "fast" | "balanced" | "detailed";
};
```

请求示例：

```json
{
  "query": "如何给成员配置管理员权限？",
  "scope": {
    "knowledgeBaseIds": ["kb_001"],
    "categoryIds": ["cat_permission"],
    "tagIds": ["tag_admin"],
    "chunkTypes": ["text", "wiki"]
  },
  "mode": "balanced"
}
```

## 4. 检索接口返回

检索模块返回的是给 Agent 或大模型参考的知识素材，不直接生成最终回答。

### 4.1 返回结构

```ts
type RagRetrieveResponse = {
  /** 本次检索使用的问题。 */
  query: string;

  /** 结构化知识素材列表。 */
  contexts: RagContext[];

  /** 已整理好的上下文文本，可以直接拼进大模型 prompt。 */
  llmContext: string;

  /** 引用来源列表，用于前端展示或回答溯源。 */
  references: RagReference[];
};

type RagContext = {
  /** 当前返回素材的 ID。 */
  id: string;

  /** 来源知识库 ID。 */
  knowledgeBaseId: string;

  /** 来源知识 ID。 */
  knowledgeId: string;

  /** 来源 chunk ID。 */
  chunkId: string;

  /** 来源知识标题。 */
  title: string;

  /** 本次返回给大模型参考的正文内容。 */
  content: string;

  /** 当前素材对应的 chunk 类型。 */
  chunkType: "text" | "wiki" | "summary" | "qa";

  /** 相关性分数；数值越大表示越相关。 */
  score: number;

  /** 来源分类 ID。 */
  categoryId?: string;

  /** 来源标签 ID 列表。 */
  tagIds?: string[];

  /** 扩展信息，例如章节路径、页码、来源文件名等。 */
  metadata?: Record<string, unknown>;
};

type RagReference = {
  /** 来源知识库 ID。 */
  knowledgeBaseId: string;

  /** 来源知识 ID。 */
  knowledgeId: string;

  /** 来源 chunk ID。 */
  chunkId: string;

  /** 来源知识标题。 */
  title: string;

  /** 来源 chunk 类型。 */
  chunkType: "text" | "wiki" | "summary" | "qa";
};
```

### 4.2 返回示例

```json
{
  "query": "如何给成员配置管理员权限？",
  "contexts": [
    {
      "id": "ctx_001",
      "knowledgeBaseId": "kb_001",
      "knowledgeId": "wiki_001",
      "chunkId": "chunk_wiki_001",
      "title": "账号权限配置说明",
      "content": "管理员可以在系统设置中进入权限管理页面，为成员分配不同角色...",
      "chunkType": "wiki",
      "score": 0.86,
      "categoryId": "cat_permission",
      "tagIds": ["tag_admin", "tag_permission"],
      "metadata": {
        "sectionPath": "账号管理/权限配置"
      }
    }
  ],
  "llmContext": "[ref_1] 账号权限配置说明\n管理员可以在系统设置中进入权限管理页面，为成员分配不同角色...",
  "references": [
    {
      "knowledgeBaseId": "kb_001",
      "knowledgeId": "wiki_001",
      "chunkId": "chunk_wiki_001",
      "title": "账号权限配置说明",
      "chunkType": "wiki"
    }
  ]
}
```

## 5. 对接注意事项

- 调用方必须传入 `scope.knowledgeBaseIds`，避免检索范围不明确。
- 如果调用方不传分类、标签、知识 ID 或 chunk 类型，表示在传入的知识库范围内不做该项限制。
- 检索模块默认只返回 `available` 状态的知识素材。
- 空结果时返回 `contexts: []`、`llmContext: ""`、`references: []`。
- `llmContext` 面向大模型使用，`references` 面向引用展示和结果追踪。
